Historia de Internet y el Desarrollo Web

  • 1969: Creación de ARPANET, el precursor de Internet.
  • 1989: Tim Berners-Lee propone la World Wide Web.
  • Años 90: Popularización de navegadores como Mosaic y Netscape.
  • Presente: Expansión del desarrollo web con tecnologías avanzadas.

¿Cómo funciona Internet?

  • Conexión: Los dispositivos se conectan a través de redes (Wi-Fi, fibra óptica, etc.).
  • Protocolos: Internet utiliza protocolos como TCP/IP para garantizar la comunicación.
  • Direcciones IP: Cada dispositivo conectado tiene una dirección única.

¿Cómo funciona Internet? (parte 2)

  • DNS: Traduce nombres de dominio (e.g., www.example.com) a direcciones IP.
  • Rutas: Los datos viajan en paquetes a través de múltiples servidores para llegar a su destino.
  • Seguridad: HTTPS y otras tecnologías protegen los datos en tránsito.

Evolución de los Navegadores

  • 1993: Lanzamiento de Mosaic.
  • 1994: Netscape Navigator domina el mercado.
  • 2004: Lanzamiento de Mozilla Firefox.
  • 2008: Google Chrome redefine la navegación web.
  • Actualidad: Competencia entre Chrome, Firefox, Edge y Safari.

Cliente y Servidor

  • Cliente: Dispositivo o aplicación que realiza solicitudes (e.g., navegadores).
  • Servidor: Máquina que procesa y responde a las solicitudes del cliente.
  • Ejemplo: Cuando visitas una página web, tu navegador (cliente) solicita datos al servidor.

Protocolos: HTTP

  • HTTP (HyperText Transfer Protocol): Protocolo para la transferencia de datos en la web.
  • HTTPS: Versión segura de HTTP, con cifrado mediante SSL/TLS.
  • Otras capas:
    • DNS: Traduce nombres de dominio a direcciones IP.
    • TCP/IP: Protocolo base de comunicación en Internet.

¿Qué es GIT?

  • GIT: Sistema de control de versiones distribuido.
  • Permite a los desarrolladores colaborar y realizar seguimiento de los cambios en el código.
  • Creado por Linus Torvalds en 2005.

¿Qué es Github?

  • Github es una de las nubes disponibles para subir proyectos
  • Permite colaborar en proyectos de forma remota
  • Es de uso gratuito y tiene funcionalidades adicionales de pago

Libro de Git y Github

Para aprender a usar el control de cambios y las funcionalidades colabortivas de Github vamos a seguir algunos capítulos del libro.

Comandos básicos de GIT

  • git init: Inicializa un repositorio vacío.
  • git add: Añade cambios al área de preparación (staging).
  • git commit: Guarda los cambios en el repositorio con un mensaje descriptivo.
  • git status: Muestra el estado actual del repositorio.

Trabajo en equipo con GIT

  • git clone: Copia un repositorio remoto.
  • git pull: Actualiza el repositorio local con los cambios remotos.
  • git push: Envía los cambios locales al repositorio remoto.
  • Branching: Uso de ramas para trabajar en nuevas características sin afectar el código principal.
