    Apuntes para preparar el examen de Azure fundmentals AZ900

    Conceptos básicos

    Parte I: descripción de la nube (cloud)
    Describir la Nube

    Conceptos Clave:

    • Identificar beneficios y consideraciones del uso de servicios en la nube.
    • Describir diferencias entre categorías de servicios y tipos de computación en la nube.

    Objetivos:

    1. Identificar beneficios como alta disponibilidad, escalabilidad, elasticidad, agilidad y recuperación de desastres.
    2. Diferenciar entre gastos de capital (CapEx) y gastos operativos (OpEx).
    3. Describir el modelo basado en el consumo.
    ¿Qué es la Computación en la Nube?

    La computación en la nube permite el acceso a recursos informáticos a través de Internet sin necesidad de infraestructura local. Proveedores como Amazon Web Services, Google Cloud Platform y Microsoft Azure ofrecen servicios en un modelo de pago por uso, permitiendo flexibilidad y escalabilidad.

    Beneficios Económicos

    La nube es más económica debido a:

    • Modelo de pago por uso.
    • Eliminación de inversiones en hardware.
    • Escalabilidad según demanda.
    • Precios reducidos gracias a economías de escala y eficiencia de los proveedores.
    Tipos de Nubes
    • Nube Pública: Sin costos de capital inicial, paga solo por lo que se usa.
    • Nube Privada: Mayor control y seguridad, pero requiere inversión en hardware.
    • Nube Híbrida: Combina ambas, ofreciendo flexibilidad y control.
    Ventajas de la Computación en la Nube
    • Alta Disponibilidad: Experiencia continua sin tiempo de inactividad.
    • Escalabilidad: Vertical (agregar RAM/CPU) y horizontal (agregar máquinas virtuales).
    • Elasticidad: Recursos según demanda.
    • Agilidad: Rápida implementación y configuración de recursos.
    • Distribución Geográfica: Mejora rendimiento y acceso global.
    • Recuperación ante Desastres: Copias de seguridad y replicación de datos.
    CapEx vs. OpEx
    • CapEx: Gasto inicial en infraestructura física.
    • OpEx: Gasto en servicios actuales, sin costo inicial, pagando conforme se usa.
    Modelos de Servicios en la Nube

    IaaS (Infrastructure as a Service):

    • Agilidad y flexibilidad en la gestión de hardware.
    • Modelo basado en el consumo.

    PaaS (Platform as a Service):

    • Agilidad superior al no requerir configuración de servidores.
    • Enfocado en el desarrollo de aplicaciones.

    SaaS (Software as a Service):

    • Acceso a aplicaciones desde cualquier lugar.
    • Pago por suscripción mensual o anual.
    Computación Serverless

    Permite a los desarrolladores enfocarse en crear aplicaciones sin gestionar infraestructura. Escalable y basado en eventos.

    ¿Qué es Azure?

    Azure es una plataforma de servicios en la nube de Microsoft que permite crear, gestionar y desplegar aplicaciones en una red global utilizando herramientas y marcos familiares.

    Parte II: servicios core en la nube (cloud)
    Servicios Core en la Nube

    Objetivos:

    • Describir los servicios principales de Azure.
    • Describir los componentes básicos de la arquitectura de Azure.
    • Describir los recursos principales disponibles en Azure.
    Información General de Azure
    • Recursos: Instancias de servicios creadas, como máquinas virtuales, almacenamiento o bases de datos SQL.
    • Grupos de Recursos: Contenedores lógicos donde se implementan y administran recursos de Azure.
    • Suscripciones: Agrupan cuentas de usuario y recursos creados, con límites y cuotas.
    • Grupos de Administración: Ayudan a gestionar el acceso, la política y el cumplimiento de varias suscripciones.
    Regiones de Azure
    • Región: Área geográfica con uno o más centros de datos conectados con una red de baja latencia.
    • Regiones Especiales: Incluyen instancias aisladas para socios y agencias gubernamentales de EE. UU., y regiones en China gestionadas por 21Vianet.
    Zonas de Disponibilidad

    Centros de datos separados físicamente dentro de una región de Azure, equipados con alimentación, refrigeración y redes independientes. Están diseñados para ser límites de aislamiento y están conectados a través de redes privadas de fibra óptica de alta velocidad.

    Pares de Regiones

    Regiones emparejadas para minimizar el tiempo de inactividad y garantizar la recuperación rápida durante interrupciones extensas.

    Azure Resource Manager

    Servicio de implementación y administración que permite crear, actualizar y eliminar recursos. Proporciona una capa de administración con control de acceso, bloqueos y etiquetas para organizar los recursos.

    Suscripciones de Azure
    • Límite de Facturación: Permite crear suscripciones para diferentes requisitos de facturación con informes y facturas independientes.
    • Límite de Control de Acceso: Aplica políticas de administración de acceso a nivel de suscripción.
    Grupos de Administración de Azure

    Organizan suscripciones en contenedores y aplican condiciones de gobierno. Las suscripciones dentro de un grupo de administración heredan las condiciones aplicadas.

    Calcular
    • Máquinas Virtuales: Emulaciones de software de computadoras físicas, incluyendo procesador virtual, memoria, almacenamiento y recursos de red.
    • Contenedores: Entornos de aplicaciones virtualizados y livianos para crear y escalar rápidamente.
    • Servicio de Aplicaciones: Plataforma para crear, desplegar y escalar aplicaciones web, móviles y API.
    • Funciones de Azure: Permiten realizar trabajos en respuesta a eventos específicos sin gestionar infraestructura.
    Redes Virtuales de Azure

    Permiten que los recursos de Azure se comuniquen entre sí y con usuarios en Internet. Proporcionan funcionalidades clave como aislamiento, segmentación, enrutamiento y filtrado del tráfico de red.

    Almacenamiento de Azure
    • Disk Storage: Proporciona discos para máquinas virtuales con diferentes tamaños y niveles de rendimiento.
    • Blob Storage: Almacena grandes cantidades de datos no estructurados.
    • Azure Files: Ofrece recursos compartidos de archivos accesibles a través de protocolos estándar.
    Bases de Datos en Azure
    • Azure Cosmos DB: Servicio de base de datos multimodelo distribuido globalmente.
    • Azure SQL Database: Base de datos relacional basada en Microsoft SQL Server.
    • Azure Database for MySQL: Servicio de base de datos relacional basado en MySQL Community Edition.
    • Azure Database for PostgreSQL: Servicio de base de datos relacional basado en PostgreSQL.
    • Instancia Administrada de Azure SQL: Combina compatibilidad con SQL Server con beneficios de PaaS.
    Herramientas de Análisis de Azure
    • Azure Synapse Analytics: Servicio de análisis ilimitado que combina almacenamiento de datos y análisis de big data.
    • Azure Data Factory: Servicio de integración de datos para orquestar y automatizar el movimiento y la transformación de datos.
    • Azure Databricks: Plataforma para obtener información de datos y crear soluciones de inteligencia artificial.
    • Azure Data Lake Storage y Analytics: Depósito de almacenamiento para grandes cantidades de datos y servicio de análisis bajo demanda.
    Parte III: Principales Servicios en la Nube y Herramientas de Administración en Azure
    Principales Servicios en la Nube y Herramientas de Administración en Azure

    Objetivos:

    • Describir las soluciones principales disponibles en Azure.
    • Describir las herramientas de administración de Azure.
    Internet de las Cosas (IoT)

    IoT permite que los dispositivos recopilen y transmitan información para análisis de datos a través de sensores inteligentes.

    Centro de IoT de Azure (Azure IoT Hub):

    Servicio gestionado para la comunicación bidireccional entre dispositivos IoT y aplicaciones. Admite múltiples patrones de mensajería y permite la conexión de cualquier dispositivo.

    Azure IoT Central:

    Basado en IoT Hub, proporciona una interfaz para conectar, monitorear y administrar dispositivos IoT con facilidad.

    Azure Sphere:

    Solución integral de IoT que abarca desde hardware hasta un método seguro de envío de mensajes. Incluye MCU, un sistema operativo Linux personalizado y un servicio de seguridad (AS3).

    Servicios Cognitivos y de Machine Learning

    Azure Machine Learning:

    Plataforma para hacer predicciones con herramientas y servicios para entrenar y probar modelos. Permite implementar modelos como API para uso en tiempo real.

    Azure Cognitive Services:

    Proporciona modelos de aprendizaje automático para aplicaciones que pueden ver, oír, hablar, comprender y razonar. Accesible a través de APIs.

    Servicio de Bots de Azure:

    Plataforma para crear agentes virtuales que entienden y responden preguntas como un ser humano, facilitando tareas automatizadas y repetitivas.

    Servicios DevOps

    Azure DevOps Services:

    Conjunto de servicios que abordan cada etapa del ciclo de vida del desarrollo de software, incluyendo repositorios de código, gestión de proyectos, automatización de pipelines CI/CD, y pruebas automatizadas.

    GitHub y GitHub Actions:

    Repositorio de código fuente y plataforma de colaboración para proyectos de software. GitHub Actions permite la automatización de flujos de trabajo CI/CD.

    Azure DevTest Labs:

    Proporciona un medio automatizado para administrar la creación, configuración y desmontaje de máquinas virtuales para pruebas de software.

    Seguridad en Azure

    Azure Security Center:

    Servicio de supervisión que proporciona visibilidad de la postura de seguridad en servicios locales y en la nube. Ofrece recomendaciones de seguridad y detecta amenazas.

    Azure Sentinel:

    Sistema SIEM basado en la nube que utiliza análisis de seguridad inteligente y análisis de amenazas. Permite recopilar datos, detectar amenazas, investigar y responder a incidentes.

    Azure Key Vault:

    Servicio para almacenar y gestionar de forma segura secretos, claves de cifrado y certificados SSL/TLS. Facilita el control de acceso y la supervisión de estos recursos.

    Azure Dedicated Host:

    Proporciona servidores físicos dedicados para alojar máquinas virtuales, con visibilidad y control sobre la infraestructura del servidor.

    Herramientas de Administración de Azure

    Azure Portal:

    Interfaz web gráfica para gestionar servicios de Azure, crear nuevos servicios y configurar los existentes.

    Aplicación Móvil de Azure:

    Proporciona acceso a recursos de Azure desde dispositivos iOS y Android, permitiendo la monitorización y gestión de recursos.

    Azure PowerShell y Azure CLI:

    Herramientas de línea de comandos para ejecutar tareas de administración en Azure, disponibles para Windows, Linux y Mac.

    Plantillas ARM:

    Describen los recursos a utilizar en un formato JSON declarativo, permitiendo la creación de recursos de manera repetible y automatizada.

    Monitorización y Asesoría

    Azure Monitor:

    Plataforma para recopilar, analizar y visualizar datos de métricas y registros de todo el entorno local y de Azure.

    Azure Service Health:

    Proporciona una vista personalizada del estado de los servicios de Azure, mostrando problemas localizados y eventos de mantenimiento.

    Azure Advisor:

    Evalúa recursos de Azure y hace recomendaciones para mejorar la confiabilidad, seguridad, rendimiento y reducir costos.

    Seguridad de Red

    Azure Firewall:

    Servicio de seguridad de red administrado y basado en la nube que protege los recursos en redes virtuales de Azure. Proporciona reglas de aplicación, red y NAT.

    Protección contra DDoS de Azure:

    Ayuda a proteger recursos de Azure contra ataques DDoS, utilizando la escala y elasticidad de la red global de Microsoft para mitigar ataques antes de que afecten la disponibilidad del servicio.
